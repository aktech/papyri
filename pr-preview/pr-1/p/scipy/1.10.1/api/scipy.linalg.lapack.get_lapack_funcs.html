Several LAPACK routines work best when its internal WORK array has the optimal size (big enough for fast computation and small enough to avoid waste of memory). This size is determined also by a dedicated query to the function which is often wrapped as a standalone function and commonly denoted as ``###_lwork``. Below is an example for ``?sysv`` This example is valid syntax, but we were not able to check execution
>>> a = rng.random((1000, 1000))
... b = rng.random((1000, 1)) * 1j
... # We pick up zsysv and zsysv_lwork due to b array
... xsysv, xlwork = LA.get_lapack_funcs(('sysv', 'sysv_lwork'), (a, b))
... opt_lwork, _ = xlwork(a.shape[0])  # returns a complex for 'z' prefix
... udut, ipiv, x, info = xsysv(a, b, lwork=int(opt_lwork.real))
