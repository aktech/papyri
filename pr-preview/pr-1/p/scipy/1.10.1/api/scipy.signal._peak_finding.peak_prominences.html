Let's evaluate a second example that demonstrates several edge cases for one peak at index 5. This example is valid syntax, but we were not able to check execution
>>> x = np.array([0, 1, 0, 3, 1, 3, 0, 4, 0])
... peaks = np.array([5])
... plt.plot(x)

Note how the peak at index 3 of the same height is not considered as a border while searching for the left base. Instead, two minima at 0 and 2 are found in which case the one closer to the evaluated peak is always chosen. On the right side, however, the base must be placed at 6 because the higher peak represents the right border to the evaluated area. This example is valid syntax, but we were not able to check execution
>>> peak_prominences(x, peaks, wlen=3.1)
(array([2.]), array([4]), array([6]))
