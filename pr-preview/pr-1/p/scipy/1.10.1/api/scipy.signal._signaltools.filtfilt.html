We get a fairly clean result for this artificial example because the odd extension is exact, and with the moderately long padding, the filter's transients have dissipated by the time the actual data is reached. In general, transient effects at the edges are unavoidable. The following example demonstrates the option ``method="gust"``. First, create a filter. >>> b, a = signal.ellip(4, 0.01, 120, 0.125)  # Filter to be applied.

>>> rng = np.random.default_rng()
... n = 60

Apply `filtfilt` to `sig`, once using the Gustafsson method, and once using padding, and plot the results for comparison. This example is valid syntax, but we were not able to check execution
>>> fgust = signal.filtfilt(b, a, sig, method="gust")
